## 任务四：MQTT网关开发

### 提示词原文

设计一个MQTT协议转换网关，实现传感器数据接收、协议转换和持久化存储。要求支持多种传感器协议，能够将数据转换为统一格式后存入数据库。

### AI输出摘要

生成8个核心文件：
1. main.py：网关主程序入口
2. mqtt_client.py：MQTT客户端连接管理
3. protocol_parser.py：协议解析器
4. data_transformer.py：数据转换器
5. storage.py：数据存储模块
6. models.py：数据模型定义
7. config.py：配置文件
8. requirements.txt：依赖清单

架构特点：采用异步架构，支持多传感器并发接入，使用dataclass定义数据模型。

### 验证结果

- 环境准备：在服务器上安装Python依赖
- 代码上传：通过scp将代码上传至服务器/opt/mqtt-gateway目录
- 运行测试：执行python main.py启动网关
- 报错：StorageRecord dataclass字段顺序错误导致数据解析失败

### 问题与归因

问题：StorageRecord dataclass字段顺序错误
归因：AI生成的代码中字段定义顺序与实际数据包字段顺序不匹配，导致解析时数据错位

### 迭代提示词

修复StorageRecord dataclass字段顺序，确保与实际MQTT数据包字段顺序一致。原始字段顺序为timestamp, device_id, temperature, humidity, pressure，需要调整为device_id, timestamp, temperature, humidity, pressure。

### 代码变更

修复前：
```python
@dataclass
class StorageRecord:
    timestamp: str
    device_id: str
    temperature: float
    humidity: float
    pressure: float
```

修复后：
```python
@dataclass
class StorageRecord:
    device_id: str
    timestamp: str
    temperature: float
    humidity: float
    pressure: float
```

### 经验总结

AI生成代码必查清单：
1. 检查dataclass字段顺序是否与实际数据匹配
2. 验证类型注解是否正确
3. 确认导入语句是否完整
4. 测试边界条件处理

## 任务五：API服务开发与部署

### 提示词原文

基于任务四存储的传感器数据，开发FastAPI历史数据查询服务。要求提供RESTful API接口，支持按时间范围、设备ID等条件查询历史数据。

### AI输出摘要

生成7个文件，提供6个API接口：
1. main.py：FastAPI应用入口
2. database.py：数据库连接管理
3. models.py：SQLAlchemy模型定义
4. schemas.py：Pydantic数据校验模型
5. crud.py：数据库CRUD操作
6. api/endpoints/history.py：历史数据查询接口
7. requirements.txt：依赖清单

接口列表：
- GET /api/v1/history：查询所有历史数据
- GET /api/v1/history/{device_id}：按设备ID查询
- GET /api/v1/history/range：按时间范围查询
- GET /api/v1/history/latest：查询最新数据
- GET /api/v1/history/stats：统计数据查询
- GET /api/v1/history/export：数据导出接口

### 验证结果

部署步骤：
1. 安装依赖：pip install -r requirements.txt
2. 启动服务：uvicorn main:app --host 0.0.0.0 --port 8000
3. 访问文档：http://82.156.142.37:8000/docs

启动失败，遇到3个问题：
1. __init__.py导入错误
2. timedelta未导入
3. 字段顺序不匹配

### 问题与归因

问题1：__init__.py导入错误
归因1：AI生成的代码中api/endpoints/__init__.py文件缺失，导致模块导入失败

问题2：timedelta未导入
归因2：代码中使用了datetime.timedelta但未导入

问题3：字段顺序不匹配
归因3：与任务四相同问题，API响应模型字段顺序与数据库模型不一致

### 迭代提示词

第一轮修复：
补充api/endpoints/__init__.py文件，导入timedelta模块，修复所有导入错误。

第二轮修复：
统一API响应模型与数据库模型的字段顺序，确保device_id在timestamp之前。

### 修复内容

修复1：添加api/endpoints/__init__.py
```python
from .history import router
```

修复2：导入timedelta
```python
from datetime import datetime, timedelta
```

修复3：统一字段顺序
```python
class HistoryResponse(BaseModel):
    device_id: str
    timestamp: datetime
    temperature: float
    humidity: float
    pressure: float
```

### 验证步骤

1. 重新启动服务：uvicorn main:app --host 0.0.0.0 --port 8000
2. 访问Swagger文档：http://82.156.142.37:8000/docs
3. 测试接口：
   - GET /api/v1/history
   - GET /api/v1/history/{device_id}
   - GET /api/v1/history/range?start=2024-01-01&end=2024-12-31
4. 验证响应数据格式正确

### 经验总结

AI常见错误清单：
1. __init__.py文件缺失导致模块导入失败
2. 导入语句遗漏（如timedelta）
3. 数据模型字段顺序不一致
4. 类型注解与实际数据不匹配

高效迭代技巧：
