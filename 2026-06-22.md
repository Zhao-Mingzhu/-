## 任务一：SSH 远程登录与免密文件传输

### 提示词原文
这些上传中是不是可以提前设置，使得不需要每次都输入密码...我甚至可以在服务器中设置文件夹来接受这些任务的操作...对于从服务器下载文件到 Windows 电脑，例如 scp root@82.156.142.37:/root/server.log 这样可以吗？

### AI 输出摘要
- 免密配置：使用 ssh-keygen 生成本地密钥对，通过 ssh-copy-id 将公钥发送至服务器
- 路径优化：本地使用相对路径（资源管理器地址栏输入 cmd），服务器使用绝对路径
- 目录规划：建议在 /opt 下建立专属接收文件夹（如 /opt/Task1），符合 FHS 规范

### 验证结果
- 成功生成密钥并实现 SSH 免密登录
- 在服务器建立 /opt/Task1，实现本地文件上传与服务器文件下载
- 注意：Windows 下需确保 .ssh 文件夹权限正确设置（仅当前用户可访问）

### 问题与归因
- 问题：初次使用每次传输都需输入密码
- 归因：未配置 SSH 密钥认证；缺乏统一文件管理规范

### 迭代提示词

## 任务二：Nginx 进程管理与 Linux 权限体系测试

### 提示词原文
进程管理 systemctl status/start/stop/restart nginx...创建一个新的用户Tester,密码123456来进行进程管理的具体操作。

### AI 输出摘要
- 创建普通用户 Tester，加入 wheel 组赋予 sudo 权限
- 使用 systemctl 启停服务，演示权限拦截机制
- 区分瞬时执行命令与分页器模式（status 需按 q 退出）

### 验证结果
- 成功创建 Tester 用户
- systemctl restart nginx 报错：Unit not found（CentOS 7.6 默认源无 Nginx）
- 安装 EPEL 源后成功安装 Nginx
- 权限验证：
  - 无 sudo：Access denied（符合预期）
  - 有 sudo：操作成功
- 注意：kill -9 强制终止后，systemd 可能自动重启 Nginx（与 systemctl stop 不同）

### 问题与归因
- 问题1：Failed to restart nginx.service: Unit not found
- 归因1：CentOS 7.6 默认软件源不包含 Nginx，需添加 EPEL 源或 Nginx 官方源
- 问题2：普通用户无法管理系统服务
- 归因2：Linux 权限隔离机制，需 sudo 提权

### 迭代提示词
我的里面显示Failed to restart nginx.service: Unit not found...下面应该怎么进行这个任务测试呢？

## 任务三：环境配置与最小权限实践

### 提示词原文
对于任务中的环境配置，我应该也是在/opt里面运行才能够不影响我后续任务的操作吧...上述是不是最好在Tester用户里面弄，还是原始的

### AI 输出摘要
- 在 /opt 下创建 /opt/tester_env
- 正确流程：root 创建目录 -&gt; chown 移交给 Tester -&gt; Tester 用户配置环境
- 环境变量写入 ~/.bashrc，实现用户级隔离

### 验证结果
- 掌握 sudo chown -R Tester:Tester /opt/tester_env 权限移交
- 在 Tester 用户下完成环境配置，无需重复 sudo
- 环境变量写入 ~/.bashrc

### 问题与归因
- 问题：在 root 下配置环境的风险
- 归因：root 操作易误删系统文件、污染环境变量、安全风险高
- 修正理解：并非完全拒绝 root，而是 root 仅用于创建系统目录，后续操作移交普通用户

### 迭代提示词
上述是不是最好在Tester用户里面弄，还是原始的
